Front-end Installation

Open this link to get the latest version of uno.js Library. You can download and attach the minified version of uno.js from following link. It is recommanded to download the file and attach in your side due to version change.

Include the script file in head section of your web page for the better performance. 

<script src="path/to/file/uno.min.js"></script>

Version Requirements

Be aware about the version of the source file. Features in the library are different when it comes to latest versions. This is much more important for the uno.js client program as well.

Once the browser connected to the controller, uno.js will check the version of the client and return a warning to the browser console if there is any mismatch.

Web Serial API

Since uno.js connects with the microcontroller using serial communication, your browser should have the serial communication feature. You can check the availability by checking navigator Web APIs. 

if ("serial" in navigator) {
  // The Web Serial API is supported.
}

So, make sure your browser is up to date. Also you can check the browser compatibility in MDN Docs.

Secure Context

Even if you have an updated browser, Web Serial API may not be available for unsecure context such as http:// or file://. Therefore, local development should hosted using a software like XAMPP or WAMP. Other than that, for all the secured websites in the internet would have access to this feature.

Warning for the secure context will appear in the console, that does not have access to Web Serial API to use uno.js, after you attach the script file.

Baud Rate

While uno.js communicates with the controller, using a baud rate such as 9,600 is not enough. The script may slow down due to low baud rate. uno.js library uses 2,000,000 baud rate for better performance. Those values were already defined in the library and client program as well. Therefore, changing those values may effect to your front-end program.
